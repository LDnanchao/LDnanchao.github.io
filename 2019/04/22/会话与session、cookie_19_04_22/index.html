会话与session、cookie

Veröffentlicht am 2019-04-22

会话：在计算机术语中，会话是指一个终端用户与交互系统进行通讯的过程，比如从输入账户密码进入操作系统到退出操作系统就是一个会话过程。会话较多用于网络上，TCP的三次握手就创建了一个会话，TCP关闭连接就是关闭会话。

有状态会话：客户端第一次访问服务器之后，再次访问服务器，服务器可以识别到这两次请求是同一个用户的行为。登录——主页

无状态会话：多次访问服务器，对服务器而言，每次访问都是一个新用户

会话：打开浏览器，访问服务器，关闭浏览器

请求：打开浏览器输入地址，回车，刷新

HTTP是无状态协议。每次访问服务器后，服务器和客户端就断开连接。

同时对于服务器而言，每一个浏览器都是一个用户对象，因为发来的请求头包含的sessionID不一致，而服务器是通过sessionID来识别用户

为了保存用户的状态，引入了一些的新技术

  • hidden
  • cookie
  • session
  • url

cookie：

客户端技术

程序把每一个用户的数据以cookie的形式保存在浏览器中，当用户使用浏览器访问服务器时，cookie传递服务器，服务器就能获取cookie的数据

cookie包含在请求和响应信息中，可以被本地保存，

setmaxage，控制cookie的保存时间

0，不保存，同时本地也不保存，和删除一致

单位 s

例 cookie.setmaxage(24*3600) 时间一天

cookie默认不支持中文,请不要保存中文信息

同时cookie的不安全，cookie会将信息保存在本地，值保存一些不敏感信息

session

session保存会话技术的一种，保存在服务器。一个浏览器独占一个session。

可以将一些隐私数据保存在session对象中，来达到会话状态的管理

session用法

public boolean isNew()
如果客户端还不知道该 session 会话，或者如果客户选择不参入该 session 会话，则该方法返回 true。

销毁session

public void invalidate()
该方法指示该 session 会话无效，并解除绑定到它上面的任何对象。

访问服务器，从服务器获得session对象，如果获取不到，则创建一个新的session

将session的ID获取到并保存在cookie，将cookie发回客户端

下次访问，将cookie发送给服务器，取出ID并匹配session对象
